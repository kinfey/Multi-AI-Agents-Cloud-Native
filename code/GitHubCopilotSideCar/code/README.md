# Blog Agent - Kubernetes Dual-Sidecar Pattern

Based on the `prototype/sidecar` architecture, deploys the GitHub Copilot Blog Agent with a K8s **dual-sidecar** pattern.
- **Sidecar 1 (copilot-agent)**: Uses GitHub Copilot SDK to generate content
- **Sidecar 2 (skill-server)**: Skill management service, serves SKILL.md
- **Main (blog-app)**: Nginx web viewer for results

## Architecture

```
┌──────────────────── Pod: blog-agent ────────────────────────┐
│                                                              │
│  ┌─────────────────┐ ┌──────────────────┐ ┌──────────────┐  │
│  │  blog-app        │ │  copilot-agent   │ │  skill-server│  │
│  │  (Main)          │ │  (Sidecar 1)     │ │  (Sidecar 2) │  │
│  │                  │ │                  │ │              │  │
│  │  Nginx + Monitor │ │  FastAPI +       │ │  FastAPI     │  │
│  │                  │ │  Copilot SDK +   │ │  Skill API   │  │
│  │  Port: 80        │ │  Node.js         │ │              │  │
│  │                  │ │  Port: 8001      │ │  Port: 8002  │  │
│  │                  │ │                  │ │              │  │
│  │  read /blog/ ←───┼─┤ write blog/      │ │ sync skill/  │  │
│  │                  │ │ read skill/ ←────┼─┤              │  │
│  └────────┬─────────┘ └──────┬───┬───────┘ └──────┬───────┘  │
│           │                  │   │                │          │
│           └── blog-data ─────┘   └─ skills-shared ┘          │
│              (emptyDir)            (emptyDir)                │
│                                                              │
│  ConfigMap: blog-agent-skill (SKILL.md) → skill-server       │
│  Secret: blog-agent-secret (COPILOT_GITHUB_TOKEN)            │
└──────────────────────────────────────────────────────────────┘
```

**Container Responsibilities:**
- **blog-app (Main Container)**: Nginx server, displays blog content generated by sidecars, reverse-proxies Copilot Agent API and Skill API, monitors blog directory status
- **copilot-agent (Sidecar 1)**: Runs FastAPI application, integrates GitHub Copilot SDK + Node.js, reads skill files synced by skill-server, generates blog files and writes to shared volume
- **skill-server (Sidecar 2)**: Runs FastAPI application, manages SKILL.md files, reads skill definitions from ConfigMap and syncs to shared volume, provides skill query API

**Data Flow:** ConfigMap → Skill Server syncs skill files → Copilot Agent reads skills and generates blog → writes to shared volume → App main container reads and serves

## Prerequisites

- Docker Desktop (macOS/Windows) or Docker Engine (Linux)
- kubectl
- kind
- GitHub Copilot Token

> The Makefile auto-detects host architecture: `x86_64 → linux/amd64`, `arm64/aarch64 → linux/arm64`.

### macOS

```bash
brew install kubectl kind
```

Or via Makefile:

```bash
make install-tools-macos
```

### Linux

Make sure Docker Engine is installed and running.

```bash
make install-tools-linux
```

> Binaries are installed to `/usr/local/bin` and require `sudo`.

## Quick Start

### 1. Set Token

Edit `k8s/secret.yaml` and replace `YOUR_COPILOT_GITHUB_TOKEN_HERE` with your actual token.

Or use command line:

```bash
make set-token
```

### 2. One-Click Deploy

```bash
make up
```

> `make up` will automatically: check dependencies → create kind cluster → build images → load images → deploy K8s resources

### 3. Verify

```bash
# Check status
make status

# Environment check
make doctor

# Smoke test
make smoke
```

### 4. Access

```bash
make port-forward
```

Access the following URLs:

| URL | Description |
|------|------|
| `http://localhost:8080/` | Blog Viewer homepage (Main App) |
| `http://localhost:8080/blog/` | Browse generated blog files |
| `http://localhost:8080/agent/` | Copilot Agent API (reverse proxy to Sidecar 1) |
| `http://localhost:8080/agent/health` | Copilot Agent health check |
| `http://localhost:8080/skill/` | Skill Server API (reverse proxy to Sidecar 2) |
| `http://localhost:8080/skill/skills` | View available skill list |
| `http://localhost:8001/` | Copilot Agent API (direct access to Sidecar 1) |
| `http://localhost:8002/` | Skill Server API (direct access to Sidecar 2) |

### 5. Generate a Blog Post

Once the environment is running and port-forwarding is active, send a `POST` request to the Copilot Agent's `/task` endpoint to generate a blog post.

**Via reverse proxy (recommended):**

```bash
curl -X POST http://localhost:8080/agent/task \
  -H "Content-Type: application/json" \
  -d '{"task": "Write a blog post about Kubernetes sidecar patterns", "user_id": "my-user"}'
```

**Via direct sidecar access:**

```bash
curl -X POST http://localhost:8001/task \
  -H "Content-Type: application/json" \
  -d '{"task": "Write a blog post about Kubernetes sidecar patterns"}'
```

**Request body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `task` | string | Yes | The blog topic or writing instructions |
| `user_id` | string | No | User identifier (defaults to `"anonymous"`) |

**What happens behind the scenes:**
1. The Copilot Agent creates a session using GitHub Copilot SDK (model: `claude-sonnet-4.5`)
2. It reads the SKILL.md guidelines synced by the Skill Server sidecar
3. DeepSearch is used to research the topic thoroughly
4. A Markdown blog file is generated and saved to the shared `blog/` volume as `blog-YYYY-MM-DD.md`

**View generated blogs:**

| Action | URL / Command |
|--------|---------------|
| Browse in browser | `http://localhost:8080/blog/` |
| List via API | `curl http://localhost:8080/agent/blogs` |
| Download a specific file | `curl http://localhost:8080/agent/blog/blog-2026-02-25.md` |

> **Note:** Blog generation may take a few minutes depending on the topic complexity. The `/task` endpoint will return the generated content along with `blog_path` and `download_url` in the response.

### 6. Cleanup

```bash
make down          # Delete K8s resources
make cluster-down  # Delete kind cluster
```

## Sidecar Pattern Overview

Comparison with `prototype/sidecar` Hello World example:

| Feature | Hello World Sidecar | Blog Agent (Dual Sidecar) |
|------|--------------------|--------------------|
| Main Container | Nginx (static page) | Nginx (Blog Viewer + reverse proxy) |
| Sidecar 1 | Alpine (periodic write) | FastAPI + Copilot SDK (AI content generation) |
| Sidecar 2 | — | FastAPI Skill Server (skill management) |
| Shared Volumes | 1 emptyDir | 2 emptyDirs (blog-data + skills-shared) |
| Data Flow | Sidecar → Main | Skill → Agent → Main |
| Purpose | Demo sidecar writing logs | Copilot Agent + Skill Server generating blogs |

## Manual Deployment

```bash
# Build images
docker build --platform linux/arm64 -t blog-agent-main:latest ./app
docker build --platform linux/arm64 -t blog-agent-copilot:latest ./sidecar
docker build --platform linux/arm64 -t blog-agent-skill:latest ./skill-sidecar

# Load into kind
kind load docker-image blog-agent-main:latest --name blog-agent-demo
kind load docker-image blog-agent-copilot:latest --name blog-agent-demo
kind load docker-image blog-agent-skill:latest --name blog-agent-demo

# Deploy
kubectl apply -f k8s/configmap.yaml
kubectl apply -f k8s/secret.yaml
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml

# Verify
kubectl get pods -l app=blog-agent
kubectl port-forward svc/blog-agent-svc 8080:80 8001:8001 8002:8002
```
